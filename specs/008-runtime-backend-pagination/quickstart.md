# Quickstart: 验证后端分页功能

本指南用于手动验证 Runtime App 的后端分页功能。

## 前置条件

1.  启动后端服务：
    ```bash
    make run-backend
    ```
2.  启动前端服务：
    ```bash
    make run-frontend
    ```
3.  确保数据库中存在大量数据（建议 > 50 条）。如果没有，可以使用脚本生成：
    ```bash
    uv run scripts/generate_test_accounts.py --count 60
    ```

## 验证步骤

### 1. 验证默认加载
1.  进入 Runtime App，导航到 "Account" (或其他有数据的对象) 列表页。
2.  **预期**:
    - 列表默认显示前 50 条记录。
    - 表格底部的分页控件显示 "1–50 of 60" (假设总数为 60)。

### 2. 验证翻页
1.  点击分页控件的 "下一页" (Next Page) 按钮 (`>`).
2.  **预期**:
    - 表格显示加载状态 (Loading spinner)。
    - 列表刷新，显示第 51-60 条记录。
    - 分页控件显示 "51–60 of 60"。
    - "下一页" 按钮变为禁用状态。

### 3. 验证每页数量调整
1.  在分页控件的 "Rows per page" 下拉框中选择 "10"。
2.  **预期**:
    - 列表刷新，显示第 1-10 条记录。
    - 分页控件显示 "1–10 of 60"。
    - 总页数变为 6 页。

### 4. 验证 API 响应
1.  打开浏览器开发者工具 (F12) -> Network 面板。
2.  执行上述翻页操作。
3.  查看 `/api/v1/data/Account` 的响应。
4.  **预期**: 响应格式为 `{"items": [...], "total": 60}`。
